# Entrepreneur Paradise — Project Guide

A single-page browser game: a 36-month entrepreneurship simulation. Deployed to
GitHub Pages (repo `Brucelin-tla/entrepreneur-paradise`).

## READ FIRST: `DESIGN.md`

`DESIGN.md` is the design north star. Before proposing or making any gameplay,
balance, scoring, or content change, check it. The core rule: **the intended
financial path (good debt → credit/banking → insurance cash value → passive
tax-free income → wealth + lifestyle) must be the OPTIMAL path; brute-force revenue
grinding must hit a ceiling.** Passive tax-free income is the crown jewel.

## Architecture (modular — do NOT recombine into one file)

- `index.html` — page shell only (~27 lines). Links the CSS and JS below.
- `css/styles.css` — all styles.
- `js/game.js` — all game logic. Entry point: `Game.init()`. Globals: `Game`, `CONFIG`.
- `config/*.json` — **source of truth for all game data** (actions, events,
  archetypes, starting positions, scoring, etc.). Change game content/balance HERE.
- `game.html` — standalone offline build (config inlined). Generated by `build.ps1`.

`loadConfig()` in `js/game.js` fetches `config/*.json` over HTTP first, and only
falls back to `window.EMBEDDED_CONFIG` when fetch fails (e.g. opened via file://).

## How to run / preview locally

This machine has **no Node, npm, or real Python** — do not use `npx`/`http-server`.
- Local server: `serve.ps1` (PowerShell static server). `.claude/launch.json` "game"
  config runs it on port 3000. Use the preview tool's `preview_start` with name "game".
- Always preview over HTTP (not file://) so `fetch` of config works.

## How to build / deploy

- **ALWAYS on every release/push: bump version + timestamp + add an accumulating patch
  note.** Add a new entry at the TOP of the `PATCH_NOTES` array in `js/game.js`
  (`{v,d:'YYYY-MM-DD HH:MM',n:[...]}`, newest first, short player-facing notes). The
  title screen version line and "What's New" panel derive from it automatically — never
  hand-edit the version in index.html. Keep old entries (the changelog accumulates).
  Also add a matching entry to `RELEASE_NOTES.md`. (The `/release` command does this.)
- Live site uses `index.html` + `css/` + `js/` + `config/` directly — just commit and push.
- `build.ps1` regenerates `game.html` (offline single-file). It is idempotent.
  (It previously had a bug that stacked duplicate config blobs — fixed; keep it idempotent.)

## Content / balance

- Uses the simplified action set: Marketing 7, Operations 10, Finance 12 = 29 actions.
- Progression stages: Foundation → Leverage → Wealth.

## Working preferences

- Edit the small modular files; never reintroduce a monolith.
- Tune numbers in `config/*.json`, not in code.
- After code changes, verify in the preview (serve + check console + screenshot), don't assume.
- Legacy files safe to ignore/delete: `index.html.backup`, `index_clean.html`, `deploy.zip`.
